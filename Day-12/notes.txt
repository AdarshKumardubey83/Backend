Cookies storage ka pura access server ke pass rahega aur server usme kuch bhi likh sakta hai, aur usme kuch bhi padh sakta hai

jabhi token create hoga , is token ko hum set kar denge ek cookies ke andar client side pe , aur iske liye hume ek aur package ki jarurat hogi : cookie-parser

q) Hum cookies ke andar aakhir token ko rakh hi kyu rahe hai , aakhir iski jarurat kyu padi?
Ans: Agar cookies-storage ke andar token rehta hai , toh client kuch bhi request karega server pe , toh server sidha cookies-storage mein se token nikalke padh leta hai, toh hume baar baar manual code nhi likhna padta hai ki server kaise token nikalega.
Server waha khud hi cookies-storage se token padh leta hai, kyuki baki storage ka access server ke pass nhi rehta hai
