Authentication system mein sabse pehle user register karta hai by giving details like name, email aur password. Server user ki information database mein store kar deta hai (usually password ko hash karke). Jab user login karta hai to server email aur password verify karta hai. Agar credentials sahi hote hain to server ek token generate karta hai (mostly JWT) aur usse user ko bhej deta hai. 
User is token ko apni side par store kar leta hai (localStorage ya cookies mein). Uske baad jab bhi user koi protected route access karta hai, to request ke saath token bhi bhejta hai. Server token ko verify karta hai, aur token ki help se server ye identify kar pata hai ki request kaun sa particular user bhej raha hai. Agar token valid hota hai to user ko access mil jata hai, warna request reject ho jati hai.
